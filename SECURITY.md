# Security Policy

## Supported Versions

Security fixes are applied to the latest released version of `neuralspotx`
published on PyPI. Please upgrade to the latest version before reporting an
issue.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in `neuralspotx`, please
report it privately. **Do not open a public GitHub issue for security
reports.**

Preferred reporting channels:

1. **GitHub private security advisories** — use the
   [Report a vulnerability](https://github.com/AmbiqAI/neuralspotx/security/advisories/new)
   form on this repository.
2. **Email** — contact [support@ambiq.com](mailto:support@ambiq.com) with the
   subject line `neuralspotx security report`.

Please include:

- a description of the issue and its potential impact,
- the affected version(s),
- steps to reproduce or a proof of concept, and
- any suggested remediation, if known.

## Disclosure Process

- We will acknowledge receipt of your report.
- We will investigate and work on a fix, keeping you informed of progress.
- We will coordinate a disclosure timeline with you and credit reporters who
  wish to be acknowledged.

Thank you for helping keep `neuralspotx` and its users safe.
